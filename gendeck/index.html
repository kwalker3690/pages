TL;DR

Every deck starts with a design guide — 6 files that define how a deck should look and feel. The pipeline decomposes generation into focused steps: Plan the narrative arc, Outline the content, Style Resolve the visual register, Structure the spatial composition, Layout Resolve into valid GML. Each step reads only what it needs from the guide.

The key insight from designer review: "consistency IS the design." The style resolver commits deck-wide to a register, variant family, and color strategy in one call — so every card follows the same art direction. Card groups enforce this further: one exemplar card is fully resolved, then cloned for structurally identical cards.

Not every flow needs the full pipeline. Template remix skips style/structure/resolver entirely — templates pre-bake the design. Import as-is skips guides entirely — the source screenshots ARE the design system.

What is a Design Guide?

Each guide is a declarative design system — 6 files that shape every pipeline step. Our first production guide will be Gamma Classic, representing our current stylistic approach. Prototype guides: Swiss Grid, Classic, Consultant, Minimal.

Key Concepts

Style Resolver

LLM "art director" that commits deck-wide to a register (outline vs filled), specific variants, color strategy, and prohibitions in a single prose brief. Every downstream card follows this brief.

Card Groups

Cards sharing the same structural role. Planner tags them, style resolver defines the group's look, one exemplar is fully resolved, the rest clone its GML. Consistency by construction.

Spatial Advisor

Real measurements from 11K+ Storybook renders. Tells the structurer whether content fits in a given layout variant before the resolver commits. "solidBoxes ×3: fits. stats ×3: too long."

gml-schema

Infrastructure layer (proposed). Single source of truth for all GML block definitions — tags, attributes, nesting rules. blocks.ts overlays guide preferences on top. Replaces hand-maintained tag specs.

Outline Format

HTML matching Gen3's output. Each <section> has imageLayout and group attributes, an <instructions> element with creative direction, and semantic content.

Vision-Model Graders

Swiss card, Swiss deck, Consultant card. Score 0.0–1.0 on design principles from screenshots. Catch broken renders, weak grid structure, missing evidence zones. Ignore renderer-level styling.
