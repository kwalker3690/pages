Top 3 Changes

1. Dense content examples in examples.md (+25% pass rate)

Added three new GML examples showing dense content handled within the 50/50 column system: heading + text in left column with stats in right, outlineBoxes in one column with empty opposite, dense narrative confined to one column. These gave the LLM a concrete pattern. Before: dense cards went full-width, destroying the grid. After: dense cards stayed within columns.

2. "Dense Content Strategy" in structure.md (+12.5% pass rate)

Five principles: condense don't drop, columns mandatory even for dense cards, scale contrast for hierarchy, vertical zoning within a column, three items not five.

The most impactful sub-principle: "Put the heading in one column and the data in the OTHER column." This single instruction was the most reliable Swiss pattern for dense content. It creates asymmetric density contrast that scores 0.70+ consistently.

3. Outline content calibration

Max 3 items per smart-layout, one paragraph per content block, instructions fields to guide the planner. Sweet spot: heading + one paragraph + 3 list items per card.
