Pipeline Architecture

Outline YAML → Plan (LLM: deck-level rhythm & pacing) → style.yaml (deterministic: variant preferences per guide) → Structure (LLM: spatial composition per card + spatial advisor) → Resolve (LLM: spatial description → GML) → deck.gml → Screenshots Each guide = a design system: density.yaml — word count targets planning.md — narrative arc, rhythm (tag-free) structure.md — spatial composition patterns (tag-free) blocks.ts — what's allowed (capabilities) style.yaml — visual vocabulary preferences examples.md — reference GML (show don't tell)

Architecture — Design System Compiler

Each guide is a declarative design system. The pipeline compiles these constraints into GML.

style.yaml — Visual Vocabulary

Deterministic variant preferences per guide. Swiss: 1 family, geometric only. Consultant: unlimited, full analytical palette. Replaces LLM call with config read.

Spatial Advisor

11,343 real Storybook measurements. Structure step sees cell dimensions before composing. "solidBoxes x4 in full: 18.5em × 12.2em, max ~222 chars. FITS."

Vision-Model Graders

Swiss card, Swiss deck, Consultant card. Score 0.0–1.0 on design principles. Catch broken renders, weak grid structure, missing evidence zones. Ignore renderer-level styling.

Guide Decomposition

density → planning → structure → style → examples. Each file owns one concern. Guides are composable and independently testable.
